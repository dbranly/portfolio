Q.Prescription
Projet académique · ESILV · LLM & GenAI · Jan. 2026
Système d'analyse automatique de prescriptions médicales combinant OCR (PaddleOCR) et LLM (GPT-4o). Pipeline adaptatif à 3 étapes : extraction OCR, routage intelligent selon le score de confiance multi-facteurs, puis structuration JSON via LLM Text ou LLM Vision selon la complexité du document.
3 stages
Pipeline adaptatif / Adaptive
60%
Seuil OCR → Vision / OCR threshold
Contexte & Problématique
Les prescriptions médicales présentent des défis uniques pour l'automatisation : contenu manuscrit, mise en page variable selon les établissements, présence de signatures, et terminologie médicale spécialisée. Un OCR seul ne suffit pas — il peut afficher une haute confiance sur les quelques mots imprimés tout en ratant l'intégralité du contenu manuscrit.
Solution développée
Pipeline adaptatif à 3 étapes : (1) PaddleOCR avec score de confiance multi-facteurs (base OCR 50% + densité de détection 25% + longueur de contenu 25%). (2) Routage intelligent : si confiance ≥ 60%, GPT-4o-mini structure le texte OCR en JSON (rapide et économique) ; sinon, GPT-4o Vision analyse directement l'image pour gérer le manuscrit. (3) Analyse visuelle systématique via LLM Vision pour la détection de signature. Interface Streamlit avec bibliothèque de prescriptions et export JSON.
Stack technique
Python PaddleOCR GPT-4o / GPT-4o-mini Streamlit Pydantic PyMuPDF OpenCV
